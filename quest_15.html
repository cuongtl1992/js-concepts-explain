The Problem with Uncleaned Timers

In Single-Page Applications (SPAs), when you create a timer using setInterval() without properly cleaning it up with clearInterval(), it leads to memory leaks.

Why Is This a Problem?

  • Memory Leaks: Timers keep references to their callback functions and any variables in their scope, preventing garbage collection.
  • Performance Degradation: Each uncleaned timer continues to execute in the background, consuming CPU resources.
  • Unexpected Behavior: When navigating between different views/components in an SPA, old timers continue to run, potentially causing unexpected state changes.

The Proper Solution

let counter = 0; let timer; function startCounter() { timer = setInterval(() => { console.log(counter++); }, 100); } function stopCounter() { clearInterval(timer); timer = null; } // When component unmounts or page changes stopCounter();

Best Practices

  • Always store timer IDs in variables so you can reference them later.
  • Clean up timers in component unmount methods (React), or when navigating away from pages.
  • Consider using requestAnimationFrame for animations, which automatically pauses when the tab is inactive.
  • Use WeakMap or WeakSet for complex timer management to allow garbage collection.
