Arrow Functions and 'this' Context

const button = document.getElementById('btn'); button.addEventListener('click', () => { console.log(this.id); });

Explanation

In JavaScript, the value of this depends on how a function is called:

Regular Function Arrow Function
Has its own this binding that is determined at call-time Does not have its own this binding
When used as an event handler, this refers to the element that triggered the event Captures this from its surrounding lexical context (enclosing scope)
function() { console.log(this.id); } - this would be the button element () => { console.log(this.id); } - this is the same as outside the function (likely window)

Key Takeaways

  • Arrow functions don't have their own this binding.
  • They inherit this from the parent scope at the time they're defined.
  • In the global context, this refers to the global object (window in browsers).
  • Since window doesn't have an id property by default, this.id will be undefined.
  • If you need this to refer to the event target, use a regular function instead: function() { console.log(this.id); }
