What's Happening?

This visualization demonstrates how JavaScript's event loop processes code involving closures, microtasks (Promises), and macrotasks (setTimeout).

Key Concepts:

  • Closure: The count variable is enclosed in the lexical environment of createCounter, allowing the returned methods to access and modify it even after the function has returned.
  • Event Loop Priority: JavaScript processes tasks in this order:
    1. Synchronous code
    2. All microtasks (Promises)
    3. One macrotask (setTimeout)
    4. Back to microtasks
  • Output Order: This execution model explains why the console logs appear in the order they do, despite the code's apparent structure.
