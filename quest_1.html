JavaScript Implicit Type Conversion

const a = "10"; const b = 5; const result = a - b + a; console.log(result);
Initializing variables

What's happening here?

This example demonstrates JavaScript's implicit type conversion (type coercion) in action. When performing operations with mixed types, JavaScript automatically converts types according to specific rules:

  • For subtraction (-): Non-number types are converted to numbers
  • For addition (+): If one operand is a string, the other is converted to a string and concatenation occurs

The Step-by-Step Process:

  1. We have a = "10" (string) and b = 5 (number)
  2. In a - b, the string "10" is converted to number 10 because of the - operator
  3. So 10 - 5 gives us 5 (number)
  4. Then in 5 + a, the + operator sees a string "10" as the second operand
  5. This causes the 5 to be converted to string "5"
  6. So "5" + "10" gives us "510" (string concatenation)
  7. But since JavaScript prints both string "510" and number 510 the same way, it can be confusing!
